## 2. Why 35%? — Gap Analysis

Almonzer manages Type 1 Diabetes with **Multiple Daily Injections (MDI)** — long-acting basal + rapid-acting bolus for meals and corrections. He uses a Sinocare GA-3 glucose meter (fingerstick). He trains as a hybrid athlete (calisthenics + running up to 21K). He recently moved to Malaysia with completely unfamiliar food.

The current module can log a glucose number and an insulin dose. That is **step 1 of about 12 steps** needed for clinical-grade self-management.

### Critical Gaps

1. **No carb counting integration.** Every meal requires a carb estimate → bolus calculation. Currently the nutrition module has carbs on FoodItem but no link to insulin dosing. This is the single biggest gap.

2. **No bolus calculator.** The math is: `(carbs / ICR) + ((current_glucose - target) / CF) - IOB`. None of these parameters exist in the model.

3. **No insulin-on-board tracking.** Rapid insulin lasts 3-5 hours. Taking a correction dose while IOB is still active causes stacking → dangerous hypos, especially before runs.

4. **No target range configuration.** Targets differ by time of day: tighter during sleep (80-130), looser around workouts (100-180). Currently hardcoded 70/180 lines in the chart.

5. **No exercise integration.** Almonzer identified that long runs increase insulin sensitivity for 24-48 hours and create nocturnal hypo risk. The OS should flag this automatically — a 15K run on Saturday should trigger a "reduce basal tonight" warning.

6. **No ketone tracking.** A T1D endurance athlete is at elevated DKA risk. Ketones must be tracked and flagged.

7. **No injection site rotation.** MDI users need to rotate sites to prevent lipohypertrophy (scar tissue that impairs insulin absorption).

8. **No CGM readiness.** He currently uses fingersticks but may get a CGM. The model should support it.

### 4.2 IOB Calculation Logic

Insulin On Board uses a standard exponential decay model. Different insulin types have different duration curves:

```typescript
// lib/iob.ts
interface InsulinCurve {
  durationHours: number;   // total active time
  peakHours: number;       // time to peak action
}

const CURVES: Record<string, InsulinCurve> = {
  rapid:      { durationHours: 4, peakHours: 1.5 },   // Novorapid, Humalog
  long:       { durationHours: 24, peakHours: 8 },      // Lantus, Levemir, Tresiba
  mixed:      { durationHours: 6, peakHours: 2.5 },
  correction: { durationHours: 4, peakHours: 1.5 },
};

// Standard exponential insulin activity curve
function insulinActivity(elapsedHours: number, durationHours: number, peakHours: number): number {
  if (elapsedHours >= durationHours) return 0;
  const t = elapsedHours / durationHours;
  const tp = peakHours / durationHours;
  // Bi-exponential model: rising + falling phase
  const activity = (t / tp) * Math.exp(1 - t / tp);
  return Math.max(0, activity);
}

// Calculate IOB from active doses
function calculateIOB(doses: InsulinDose[], now: Date): number {
  return doses
    .filter(d => d.type === "rapid" || d.type === "correction")
    .reduce((total, dose) => {
      const elapsed = (now.getTime() - dose.timestamp.getTime()) / 3600000;
      const curve = CURVES[dose.type];
      const remaining = dose.units * (1 - insulinActivity(elapsed, curve.durationHours, curve.peakHours));
      return total + remaining;
    }, 0);
}
```

### 4.3 Bolus Calculator Logic

```typescript
// lib/bolus.ts
interface BolusInput {
  carbs: number;
  currentGlucose: number;
  targetGlucose: number;
  icr: number;        // insulin-to-carb ratio (g carbs per unit)
  cf: number;         // correction factor (mg/dL per unit)
  iob: number;        // current insulin on board
}

interface BolusOutput {
  carbDose: number;       // units for carbs
  correctionDose: number; // units for correction
  totalDose: number;      // total suggested
  roundedDose: number;    // rounded to nearest 0.5 unit
  formula: string;        // human-readable breakdown
}

function calculateBolus(input: BolusInput): BolusOutput {
  const carbDose = input.carbs / input.icr;
  
  // Correction: how far above target, minus IOB
  const rawCorrection = (input.currentGlucose - input.targetGlucose) / input.cf;
  const correctionDose = Math.max(0, rawCorrection - input.iob);
  
  const totalDose = carbDose + correctionDose;
  const roundedDose = Math.round(totalDose * 2) / 2; // round to 0.5 units
  
  return {
    carbDose: Math.round(carbDose * 100) / 100,
    correctionDose: Math.round(correctionDose * 100) / 100,
    totalDose: Math.round(totalDose * 100) / 100,
    roundedDose,
    formula: `${input.carbs}g ÷ ${input.icr} = ${carbDose.toFixed(1)}u (carbs) + (${input.currentGlucose} - ${input.targetGlucose}) ÷ ${input.cf} = ${correctionDose.toFixed(1)}u (correction) - ${input.iob.toFixed(1)}u (IOB) = ${totalDose.toFixed(1)}u → ${roundedDose}u`
  };
}
```
